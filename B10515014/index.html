Chapter 0 :

Process and Memory :

一個xv6的程序包含了使用者空間，private的內核進程狀態。

Xv6將可用的cpu 分配給那些等待執行的程序，當程序沒有在執行時，XV6將此程序儲存於CPU暫存器並將其再下一個run時還原。有些程序會用fork 這個system call 來產生新的程序，fork指令產生的程序稱為child process，有著與calling process程序相同的記憶體內容稱作parent process。Fork 將回傳Parent & child 。再Parent中fork會回傳child的PID，child則回傳0。

exit():將程序停止執行並釋放出資源，如:記憶體、開啟的檔案。
wait():回傳當前程序的已停止之child的pid，沒有以停止的child，則會等到有為止再執行。
初始的parent & child process雖有著相同的內容，但兩只執行於不同的記憶體及暫存器，故更改其中的任一變數並不會互相影響。

exec():用一個從file system中的一個檔案載入的新記憶體映像取代calling process的記憶體。此檔案必須有個特定的格式，此格式區分了檔案的哪個部分有instruction，哪個部分有著資料和哪個部分是起始點之類的。
Xv6 指派大多的使用者記憶體。Fork指派parent 記憶體的child副本需求的記憶體，exec指派足夠的記憶體來保持可執行的檔案。

I/O and File descriptors:

read(fd, buf, n):
最多從file descriptor中讀取n 個bytes，
複製進去buffer，再回傳了以讀取的byte數。
每參考於檔案的file descriptor 會與file有offset關係。
read 會由此檔案的offset連續的讀取，當沒有byte可以讀時將回傳0作為檔案結尾的信號。

write(fd,buf,n):
從buf中寫n個byte進入 file descriptor。若寫入少於n個byte則是發生錯誤。

Chapter 1

作業系統最主要的需求就是可以支援一次進行多個活動進行，若一個程序有BUG或崩潰，作業系統必須要不讓其他程序被影響。作業系統必須滿足:多工、孤立、交互作用這三點要求。

Pipes:

Pipes是 一個小型的內核buffer，以一對的file descriptor的方式提供給Process，一個寫一個讀。

pipe():產生新的pipe與read /write檔案的file descriptor於陣列p，
fork後parent & child都會有指到pipe的file descriptor。
child 會複製 read 的結尾到 file descriptor 0，
並關閉於p中的 file descriptor 。

Section 3

Chapter 6: File system

文件系統的目的是組織與儲存數據，典型的文件系統支援用戶和程式間的數據共享，並在重新啟動後仍能保留數據。

流程

Xv6 的文件系統共六層，Block 0(包含 bootloader)。
Block 1 叫做 superblock，包含了文件系統的主要資料。
從第2個Block開始存入inode，每一個block可以儲存多個inode，
接著的Block用來存放磁碟的空間狀態圖，
剩下的大部分是用來存放資料。
最後一個Block是用來存放log紀錄。

資料結構

Buffer Cache Layer(boot)

XV6 的 buffer cache是一個雙向的linked list













 struct buf {  
     int flags;    
     uint dev;    
     uint blockno; 
     struct sleeplock lock;    
     uint refcnt;    
     struct buf *prev; // LRU cache list    
     struct buf *next;    
     struct buf *qnext; // disk queue    
     uchar data[BSIZE];  
 };

在kernel中分配了靜態數組，藉由prev來依照Block的使用頻率，按照最近使用的順序來組織結構能讓Block的讀取更加效率。
Buffer Cache Layer提供有binit，bget ，bread，BWRITE，brelse接口。

bwrite() 將buffer cache 寫入 disk








void
bwrite(struct buf *b)
{
  if(!holdingsleep(&b->lock))
    panic("bwrite");
  b->flags |= B_DIRTY;
  iderw(b);
}
brelse() 釋放sleep-lock 並將buffer放置到linked list的第一位



























// Release a locked buffer.
// Move to the head of the MRU list.
void
brelse(struct buf *b)
{
  if(!holdingsleep(&b->lock))
    panic("brelse");

  releasesleep(&b->lock);

  acquire(&bcache.lock);
  b->refcnt--;
  if (b->refcnt == 0) {
    // no one is waiting for it.
    b->next->prev = b->prev;
    b->prev->next = b->next;
    b->next = bcache.head.next;
    b->prev = &bcache.head;
    bcache.head.next->prev = b;
    bcache.head.next = b;
  }

  release(&bcache.lock);
}

iinit
















void
iinit(int dev)
{
  int i = 0;

  initlock(&icache.lock, "icache");
  for(i = 0; i < NINODE; i++) {
    initsleeplock(&icache.inode[i].lock, "inode");  //初始化inode
  }

  readsb(dev, &sb);
  cprintf("sb: size %d nblocks %d ninodes %d nlog %d logstart %d\
 inodestart %d bmap start %d\n", sb.size, sb.nblocks,
          sb.ninodes, sb.nlog, sb.logstart, sb.inodestart,
          sb.bmapstart);
}
ialloc

在硬碟中找到空的inode並傳回inode























struct inode*
ialloc(uint dev, short type)
{
  int inum;
  struct buf *bp;
  struct dinode *dip;

  for(inum = 1; inum < sb.ninodes; inum++){
    bp = bread(dev, IBLOCK(inum, sb));
    dip = (struct dinode*)bp->data + inum%IPB;
    if(dip->type == 0){  // a free inode
      memset(dip, 0, sizeof(*dip));
      dip->type = type;
      log_write(bp);   // mark it allocated on the disk
      brelse(bp);
      return iget(dev, inum);//
    }
    brelse(bp);
  }
  panic("ialloc: no inodes");
}
