  • Chapter 0

    本章說明了作業系統通過接口向用戶程序提供服務和說明了xv6系統服務：processes, memory, file descrip- tors, pipes, and file system。

    用戶可通過user space里的shell上的system call進入kernel並執行服務且返回。一个进程在user space和中互相切换。用戶程序可見的作業系統接口就是kernel提供的system call。
    1. Processes and memory：
    一個xv6進程由用戶內存空間（instruction，data，and stack）和只對kernel可見的進程狀態。

    3. Pipes

    xv6的File system提供了文件和目錄。

  • Chapter 1

    記憶體管理很重要，所以作業系統必須擁有三個要求：多路復用（multiplexing）, 隔離（isolation）和交互（interaction），本章通過描述如何組織作業系統創建第一個進程來實現以上三個要求。

    1.User mode, kernel mode, and system calls

    xv86處理器有兩種模式：User mode和 kernel mode。
进程提供了一条特殊指令，可将处理器从用户模式切换到内核模式，并在内核指定的入口点进入内核，可以
防止惡意程式跳過驗證進入內核。

    2.Kernel organization

    在xv6中，內核接口對應於作業系統接口，內核實現完整的作業系統。

    3.Process overview

    Process中將page tables映射到physical address上。p->state指示了進程的狀態：新建、準備
運行、運行、等待I/O或退出狀態中。

    4.Code: the first address space

      bootloader把xv6 kernel從硬盤載入並從entry（設置了page table ）開始運行。

    5.Code: creating the first process

    main函數通過調用userinit建立第一個進程，userinit調用allocproc分配了一個槽去初始化進程。

    6.Code: Running the first process

    mpmain調用scheduler開始運行第一個進程。

    7.The first system call: exec

    通過Initcode.S觸發exec系統調用，它能夠加載並執行一個文件。

項目三（Chapter 4 Locking ）

1.運作原理

xv6支持單核心或多核心CPU，且所有CPU共享同一個地址空間的數據，為了防止它們互相干擾和保護數據結構，xv6採用了能提供互斥功能的鎖，保證了一個CPU在某個時間能使用這個資料結構。

  • spinlock

    xv6使用一個結構體來代表專門鎖住某一數據結構的鎖。










// Mutual exclusion lock.
struct spinlock {
  uint locked;    // 是否有鎖？
  
// For debugging:
  char *name;    // 鎖的名字.
  struct cpu *cpu;    // 佔有的cpu
  uint pcs[10];    // 佔有的調用棧
                  // that locked the lock.
 };

  • xchg

    在原子操作中，xchg交換了一個暫存器的值和內存中的一個字。當locked為1時表示當前訪問的數據結構已經被鎖住，locked為0時表示未鎖住。












static inline uint
xchg(volatile uint *addr, uint newval)
{
  uint result;

  // The + in "+m" denotes a read−modify−write operand. 
  asm volatile("lock; xchgl %0, %1" :
 "+m" (*addr), "=a" (result) :
 "1" (newval) :
"cc");
  return result;
 }

  • initlock

    鎖的初始化：







 void
  initlock(struct spinlock *lk, char *name)
  {
    lk−>name = name;    //給鎖命名
    lk−>locked = 0;     //locked為0是未佔有鎖，為1時時佔有鎖
    lk−>cpu = 0;        //cpu為空
  }

  • acquire

    xv6使用函數acquire來表示獲得鎖並記錄獲得鎖CPU和棧信息：
























// Acquire the lock.
// Loops (spins) until the lock is acquired.
// Holding a lock for a long time may cause
// other CPUs to waste time spinning to acquire it.
void
acquire(struct spinlock *lk)
{
  pushcli();  //禁用中断以避免死锁。
  if(holding(lk))
    panic("acquire");

  // xchg反覆循環來檢測鎖的情況
  while(xchg(&lk−>locked, 1) != 0)
    ;

  // 告诉编译器和处理器不要移动加载或存储  
  // 通過這個点，确保关键部分的內存    
  // 佔有鎖後會發生的反應
  __sync_synchronize();

 // 記錄有關鎖定獲取的信息以進行調試
 lk−>cpu = mycpu();
 getcallerpcs(&lk, lk−>pcs);
}

  • release

    xv6使用函數release來表示解除鎖並清除調試信息：
























 // Release the lock.
 void
 release(struct spinlock *lk)
 {
   if(!holding(lk))
     panic("release");

   lk−>pcs[0] = 0;
   lk−>cpu = 0;

   // Tell the C compiler and the processor to not move loads or stores
   // past this point, to ensure that all the stores in the critical
   // section are visible to other cores before the lock is released.
   // Both the C compiler and the hardware may re−order loads and
   // stores; __sync_synchronize() tells them both not to.
   __sync_synchronize();

   // 釋放鎖，locked為0
   // This code can’t use a C assignment, since it might
   // not be atomic. A real OS would use C atomics here.
   asm volatile("movl $0, %0" : "+m" (lk−>locked) : );

   popcli();
 }

// 按照％ebp鏈記錄pcs []中的當前調用堆棧。
getcallerpcs(void *v,uint pcs[])
{
  uint *ebp;
  int i;

  ebp = (uint*)v − 2;
  for(i = 0; i < 10; i++){
     if(ebp == 0 || ebp <(uint*)KERNBASE || ebp == (uint*)0xffffffff)
       break;
     pcs[i] = ebp[1];    // 儲存%eip
     ebp = (uint*)ebp[0];// 儲存%ebp
}
for(; i < 10; i++)
  pcs[i] = 0;
}

  • holding







// 檢測CPU是否佔有鎖
int
holding(struct spinlock *lock)
{
  return lock−>locked && lock−>cpu == mycpu();
}

  • pushcli和popcli

    搶佔式內核中斷處理程序，xv6用pushcli和popcli來屏蔽中斷。pushcli主要關閉外部中斷並遞增調用popcli關閉中斷次數。















// Pushcli/popcli are like cli/sti except that they are matched:
// it takes two popcli to undo two pushcli. Also, if interrupts
// are off, then pushcli, popcli leaves them off.

void
pushcli(void)
{
  int eflags;

  eflags = readeflags();
  cli();
  if(mycpu()−>ncli == 0)
    mycpu()−>intena = eflags & FL_IF;
  mycpu()−>ncli += 1;
 }











void
popcli(void)
{
  if(readeflags()&FL_IF)
    panic("popcli − interruptible");
  if(−−mycpu()−>ncli < 0)
    panic("popcli");
  if(mycpu()−>ncli == 0 && mycpu()−>intena) 
    sti();
}
